Reports

What is a Report?

The main purpose of the reports page is to create reports. A report is made up of: parameters outlined by the user, and the data fetched from the database that fills those parameters. Abacus allows you to fully customise extensive parameters to suit and fulfill your needs in any way, shape or form.

Calendar

The main date picking feature of the interface. Clicking on the 'add a date' field on the sidebar will open up a calender. To use the calender, the user must click the dates they want their report to show, and then click done. Simple. The user must now pick what kind of range the data is going to be presented in. Clicking day will give you an interval of hours, month - an interval of days, and year - an interval of months.

This means that if you picked a date (say August 17th 2012 for example), and then clicked the day button, the presented data will show a count for each hour of that specific day.

Area/Cameras

A feature of abacus is to outline specific areas for counting. If a user only wanted to retrieve data from particular region, then they outline it in this field. Clicking the zone field offers the user the choice of which zone they would like to retrieve data from (for example, the North entrance). Abacus will then filter data depending on the zone the user wishes to see.

Chart Types

There are several different chart types to choose from. Each chart type will pull the counts from the database and display it in whatever way you would like.

This gives Abacus an edge over other basic counting systems where you have only 1 real option, however with Abacus you are able to select from an array of different formats to depict the information, certain charts will suit certain people and here at VMS we take into deep consideration every possible need of our client(s).
